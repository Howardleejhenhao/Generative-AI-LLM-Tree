# Technical Task Report: stdio MCP real handshake / tool discovery v1
**Timestamp**: 2024-04-15 23:58
**Task Objective**: Transition the `stdio` MCP client from a skeleton placeholder to a functional transport that supports real subprocess lifecycle management, JSON-RPC handshake, tool discovery, and tool invocation.

---
## 1. Technical Implementation
The implementation focused on replacing the skeleton `StdioMCPClient` with a robust version capable of interacting with actual command-based MCP servers over `stdio`.

### Key Components:
1. **Subprocess Lifecycle Management**:
   - `StdioMCPClient` now manages a `subprocess.Popen` instance.
   - It handles starting the process lazily upon the first request and ensuring it's running before subsequent calls.
   - Clean shutdown is handled via `_stop_process` (called explicitly in tests and via `__del__`).

2. **JSON-RPC Over Stdio**:
   - Implemented a synchronous request-response flow over `stdin`/`stdout`.
   - Each request has a unique `id`, and the client waits for the corresponding response, skipping intermediate notifications.
   - Strict validation ensures responses include either a `result` or an `error` field.

3. **MCP Handshake (Initialize Flow)**:
   - `_ensure_initialized` implements the `initialize` (request) and `notifications/initialized` (notification) flow as specified in the MCP protocol.
   - Server capabilities and info are captured during the handshake and surfaced through `get_server_info`.

4. **Tool Discovery and Invocation**:
   - `list_tools` sends a `tools/list` request and normalizes the JSON-RPC response into the project's internal `ToolDefinition` shape.
   - `call_tool` sends a `tools/call` request and returns a `ToolResult`.

5. **Test Strategy**:
   - Created `tree_ui/services/mcp/test_mcp_server.py`: A simple, reusable Python script that implements a minimal MCP server (with an `echo` tool) for integration testing.
   - Created `tree_ui/services/mcp/malformed_mcp_server.py`: A script that returns invalid JSON to test error handling.
