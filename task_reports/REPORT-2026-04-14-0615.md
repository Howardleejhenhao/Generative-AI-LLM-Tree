# Comprehensive Task Report: LLM-Tree v2 Tool-use Groundwork & Security Fixes

- **Timestamp**: 2026-04-14 06:20
- **Task Overview**: 
    針對 LLM-Tree v2 的內部工具呼叫（Tool-use）機制進行缺陷修正與安全性強化。重點解決 `compare_branches` 工具的越權讀取漏洞、前端 `node-chat` 在非重新整理狀態下的資料不同步問題，並建立完整的自動化測試體系以驗證 OpenAI/Gemini 路徑的正確性。

- **Actions Taken & Technical Details**:

    ### 1. 修正 `compare_branches` Workspace 越權讀取漏洞
    - **問題分析**: 原本工具直接使用 `ConversationNode.objects.get(pk=...)`，攻擊者可透過猜測 ID 讀取不屬於當前 Workspace 的對話摘要。
    - **解決方案**: 
        - 於 `tree_ui/services/tools/base.py` 擴展 `execute` 介面，引入 `context` 參數以傳遞系統層級資訊。
        - 於 `tree_ui/services/tools/branch_comparison.py` 實作驗證邏輯：若 context 中包含 `workspace` 物件，則查詢時強制加上 `workspace` 過濾條件；若無則比對兩個 Node 的 `workspace_id` 是否一致。
        - 於 `tree_ui/services/node_creation.py` 的 `generate_assistant_reply` 與 `stream_assistant_reply` 中，在執行工具前從 `parent` 節點提取 `workspace` 並封裝進 context。

    ### 2. 修正前端 `node-chat.js` 資料同步缺陷
    - **問題分析**: Streaming 結束後，後端會回傳完整的 Node 序列化資料。原本前端僅同步 `messages` 與 `attachments`，導致 `tool_invocations`（工具呼叫軌跡）在不重新整理頁面的情況下無法呈現。
    - **解決方案**: 
        - 修改 `tree_ui/static/tree_ui/js/node-chat.js` 中的 `node` callback，新增 `payload.tool_invocations = data.node.tool_invocations || [];`。
        - 此舉確保了 `renderTranscript()` 能夠即時抓取最新工具呼叫紀錄並正確渲染出工具區塊。

    ### 3. 補強測試體系 (Regression & Provider Tests)
    - **問題分析**: 原本缺乏針對工具註冊、執行與持久化的自動化驗證，且舊有測試的 mock 格式與現行 `ProviderDelta` 物件不相符。
    - **解決方案**:
        - 在 `tree_ui/tests.py` 新增 `ToolUseTests` 類別。
        - **安全性測試**: 驗證 `BranchComparisonTool` 在跨 Workspace 查詢時會正確回傳錯誤而非資料。
        - **持久化測試**: 驗證工具呼叫後，`ToolInvocation` 模型會正確記錄 `invocation_payload` 與 `result_payload`。
        - **Provider 解析測試**: 針對 OpenAI 與 Gemini 不同的 JSON 回傳結構，分別測試 `_extract_tool_calls` 邏輯，確保 args 正確轉換為 dictionary。
        - **Streaming 事件測試**: 模擬包含 `tool_call` 的 SSE 流，驗證前端能收到對應的事件。
    - **測試修復**: 全面將 `WorkspaceGraphViewTests` 中原本回傳 `str` 的 `mock_stream_text` 修正為回傳 `ProviderDelta` 物件，解決了 5 個 Error 與 2 個 Failure。
