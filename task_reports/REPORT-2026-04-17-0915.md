# Technical Task Report: Branch Comparison Inspector
**Timestamp**: 2026-04-17 09:15
**Task Objective**: Implement a branch comparison inspector so users can compare two nodes/branches inside the workspace and understand how they differ.

---
## 1. Technical Implementation
The branch comparison inspector allows users to select two nodes in the workspace graph and view a side-by-side comparison of their respective branches. This strengthens the graph-first identity of LLM-Tree v2 by making cross-branch differences explicit and useful.

### Backend Changes:
- **`tree_ui/views.py`**: Added `compare_nodes` view which accepts two node IDs and returns a detailed comparison payload. This includes node metadata (title, provider, model, routing mode/decision), branch-level statistics (depth, total tool count), and a preview of the latest assistant output.
- **`tree_ui/urls.py`**: Registered the `api/workspaces/<slug:slug>/nodes/compare/` endpoint.
- **`tree_ui/tests.py`**: Added `BranchComparisonViewTests` to verify the new endpoint's functionality, including workspace boundary checks and input validation.

### Frontend Changes:
- **`tree_ui/templates/tree_ui/index.html`**:
    - Added a "Compare" button to the node selection flags in the workspace toolbar.
    - Added a `compare-dialog` modal to display the comparison results.
    - Added a hidden data element for the comparison API URL.
- **`tree_ui/static/tree_ui/js/app.js`**:
    - Implemented `isCompareMode` state to handle the two-step node selection process.
    - Added `handleNodeCompare` to toggle comparison mode and update button state/text.
    - Modified `updateSelection` to trigger comparison when a second node is selected while in comparison mode.
    - Implemented `performComparison` to fetch data from the backend.
    - Implemented `renderComparison` to dynamically build the side-by-side comparison UI.
- **`tree_ui/static/tree_ui/css/app.css`**:
    - Added comprehensive styles for the comparison dialog, grid layout, and stat blocks to ensure readability and a polished aesthetic.

    def get_lineage_info(node):
        lineage = _build_lineage(node)
        return {
            "depth": len(lineage),
            "titles": [n.title for n in lineage],
            "total_tool_count": sum(n.tool_invocations.count() for n in lineage),
        }

    data_a["lineage"] = get_lineage_info(node_a)
