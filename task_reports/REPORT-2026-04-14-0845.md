# Technical Task Report: MCP Source Management UI
**Timestamp**: 2026-04-14 08:45
**Task Objective**: Implement a management UI for MCP sources, providing a user-facing interface to list, create, edit, and delete tool sources without using the Django shell.

---
## 1. Technical Implementation
We have implemented a complete management surface for MCP sources using Django Templates and Forms.

1.  **Form & Validation**: Created `MCPSourceForm` in `tree_ui/forms.py`.
    *   Supports all `MCPSource` fields: `name`, `source_id`, `source_type`, `is_enabled`, `description`.
    *   Introduced `config_json` CharField (textarea) to handle the `config` JSONField.
    *   Implemented `clean_config_json` to perform JSON validation, preventing invalid configurations from being saved.
2.  **Views & Routing**: Added 4 new views in `tree_ui/views.py` and corresponding URLs in `tree_ui/urls.py`:
    *   `mcp_source_list`: Displays all registered sources with status and transport info.
    *   `mcp_source_create`: Form-based creation of new sources.
    *   `mcp_source_edit`: Form-based editing of existing sources.
    *   `mcp_source_delete`: POST-only deletion with confirmation in the template.
3.  **Templates & UI Consistency**: Created two new templates that reuse the workspace shell structure:
    *   `mcp_source_list.html`: A table-based view showing source metadata.
    *   `mcp_source_form.html`: A clean form interface for adding/editing sources.
    *   Integrated "MCP Settings" links into the Workspace Graph (header actions) and Node Chat (header links) for easy access.
4.  **Verification**: Added `MCPSourceManagementTests` in `tree_ui/tests.py`, covering the full CRUD lifecycle and error handling for invalid JSON.

def clean_config_json(self):
    data = self.cleaned_data['config_json']
    if not data:
        return {}
    try:
        return json.loads(data)
    except json.JSONDecodeError:
        raise forms.ValidationError("Invalid JSON format.")
